# 智能睡眠唤醒闹钟 - Product Requirement Document

## Overview
- **Summary**: 一款基于智能手表睡眠监测数据的智能闹钟移动应用Demo，通过AI算法实时分析用户睡眠状态，在睡眠恢复效果最佳且处于适合醒来的阶段（浅睡期）自动响铃，帮助用户减少醒来后的疲惫感，提升起床体验。
- **Purpose**: 解决固定时间闹钟可能在深度睡眠阶段唤醒用户导致的疲惫、头晕、注意力下降等问题，充分利用智能手表的睡眠监测数据，实现"在对的时间醒来"。
- **Target Users**: 拥有智能手表、注重睡眠质量、希望提升起床体验的人群。

## Goals
- 实现用户登录注册与个性化数据云端保存
- 模拟智能手表睡眠数据实时读取与展示
- 实现基于睡眠阶段分析的智能唤醒算法
- 提供简约、高级、易用的移动端交互界面
- 建立个性化睡眠模型雏形，展示"千人千面"的策略

## Non-Goals (Out of Scope)
- 真实的智能手表硬件连接与数据通信（Demo阶段使用模拟数据）
- 完整的后端服务与数据库（Demo阶段使用本地存储 + Mock数据）
- 医疗级别的睡眠诊断与建议
- 多平台原生应用开发（Demo为移动端Web应用，可在手机浏览器运行）
- 实际的闹钟响铃硬件控制
- 社交分享、社区等非核心功能

## Background & Context
- 当前大多数闹钟只能按固定时间响铃，无法适配用户当日的实际睡眠状态
- 智能手表已能持续监测心率、体动、血氧、睡眠阶段等数据，但未充分用于智能唤醒
- 睡眠科学表明：在浅睡眠阶段醒来，人体更容易恢复清醒，减少睡眠惰性
- Demo阶段重点展示产品核心逻辑与交互体验，硬件连接与后端服务可后续迭代

## Functional Requirements
- **FR-1**: 用户账号系统 - 支持注册、登录、退出，个性化数据本地持久化
- **FR-2**: 设备连接模拟 - 展示与智能手表的连接状态，模拟实时睡眠数据传输
- **FR-3**: 睡眠数据实时展示 - 可视化展示心率、体动、睡眠阶段等实时数据
- **FR-4**: 智能闹钟设置 - 用户可设置唤醒时间窗口（最早/最晚唤醒时间）
- **FR-5**: AI睡眠分析算法 - 基于睡眠阶段数据计算最佳唤醒时机
- **FR-6**: 智能唤醒触发 - 在最佳时机触发闹钟响铃与唤醒界面
- **FR-7**: 睡眠报告 - 展示当晚睡眠分析结果与历史睡眠记录
- **FR-8**: 个性化模型 - 基于历史数据展示用户睡眠规律与个性化建议

## Non-Functional Requirements
- **NFR-1**: 界面设计 - 简约、高级、美观，深色主题为主，适合夜间使用
- **NFR-2**: 交互体验 - 逻辑简单直观，操作步骤少于3步即可完成核心功能设置
- **NFR-3**: 响应速度 - 页面切换响应时间 < 300ms，数据更新流畅无卡顿
- **NFR-4**: 移动端适配 - 完美适配主流手机屏幕尺寸（360px - 430px宽度）
- **NFR-5**: 可访问性 - 文字清晰可读，按钮足够大，适合夜间操作

## Constraints
- **Technical**: 
  - 使用 React + TypeScript + Vite 构建移动端Web应用
  - Demo阶段使用本地存储（localStorage）持久化数据
  - 睡眠数据使用Mock数据模拟，不连接真实硬件
  - 不引入重型UI库，保持轻量与定制化
- **Business**: 
  - 作为Demo展示核心创意与交互
  - 开发周期短，聚焦核心功能体验
- **Dependencies**: 
  - 浏览器Web Audio API（闹钟音效）
  - localStorage（数据持久化）
  - Canvas/SVG（睡眠数据可视化）

## Assumptions
- 用户使用现代手机浏览器访问（支持ES6+、Web Audio API）
- Demo阶段用户理解数据为模拟数据
- 唤醒算法基于通用睡眠科学原理，非医疗级精准
- 移动端竖屏使用为主

## Acceptance Criteria

### AC-1: 用户注册登录
- **Given**: 用户首次打开应用
- **When**: 用户填写邮箱/用户名和密码并点击注册
- **Then**: 账号创建成功，自动登录并进入主界面，数据保存在本地
- **Verification**: `programmatic`
- **Notes**: 支持记住登录状态，下次打开自动登录

### AC-2: 设备连接状态展示
- **Given**: 用户已登录
- **When**: 进入睡眠监测页面
- **Then**: 显示手表连接状态（已连接/未连接），并有模拟的实时数据波动效果
- **Verification**: `human-judgment`

### AC-3: 睡眠数据实时可视化
- **Given**: 手表已"连接"且处于睡眠监测中
- **When**: 查看睡眠监测页面
- **Then**: 实时展示心率数值、睡眠阶段（深睡/浅睡/REM/清醒）、体动等数据，并有动态变化效果
- **Verification**: `human-judgment`

### AC-4: 智能闹钟时间窗口设置
- **Given**: 用户在闹钟设置页面
- **When**: 用户设置最早唤醒时间和最晚唤醒时间
- **Then**: 设置成功保存，主界面显示下次唤醒时间窗口
- **Verification**: `programmatic`

### AC-5: AI智能唤醒时机计算
- **Given**: 用户已设置唤醒时间窗口，且处于睡眠监测中
- **When**: 当前时间进入唤醒窗口，且睡眠阶段处于浅睡期
- **Then**: 系统判定为最佳唤醒时机，触发闹钟
- **Verification**: `programmatic`
- **Notes**: 若整个窗口内均为深睡，则在窗口结束时唤醒

### AC-6: 闹钟响铃与唤醒界面
- **Given**: 到达最佳唤醒时机
- **When**: 闹钟触发
- **Then**: 显示唤醒界面，播放逐渐增强的闹铃音效，提供"停止"和"贪睡"按钮
- **Verification**: `human-judgment`

### AC-7: 睡眠报告生成
- **Given**: 用户完成一次睡眠并关闭闹钟
- **When**: 进入睡眠报告页面
- **Then**: 展示当晚睡眠总时长、各阶段占比、睡眠质量评分、最佳唤醒时机说明
- **Verification**: `human-judgment`

### AC-8: 历史记录与个性化
- **Given**: 用户有多次睡眠记录
- **When**: 进入历史/个人中心页面
- **Then**: 展示历史睡眠记录列表，以及基于历史数据的个性化睡眠规律总结
- **Verification**: `human-judgment`

### AC-9: 界面设计品质
- **Given**: 用户在任意页面
- **When**: 浏览和操作应用
- **Then**: 界面呈现简约高级的视觉风格，深色主题舒适不刺眼，动效流畅自然
- **Verification**: `human-judgment`

### AC-10: 移动端适配
- **Given**: 使用不同尺寸的手机浏览器访问
- **When**: 浏览所有页面
- **Then**: 布局自适应，无横向滚动，元素不重叠，文字清晰可读
- **Verification**: `human-judgment`
