# 睡眠页面自检与优化 Spec

## Why
当前睡眠页面存在多个问题：数据与UI显示不一致、缺少空状态处理、周/月视图数据逻辑错误、硬编码数据过多、无障碍问题等，需要全面修复和优化。

## What Changes

### 问题清单

**1. 数据一致性问题**
- [ ] "今天"标签为静态文本，未根据实际数据日期动态显示
- [ ] Mock数据睡眠时间（22:00-02:00）可能跨天，导致显示的日期与"今天"不匹配
- [ ] 环形图只显示深睡/浅睡/REM，缺少清醒部分

**2. 硬编码数据问题**
- [ ] "平均睡眠血氧 96%" 为硬编码，应从数据计算
- [ ] "深睡连续性 100分" 为硬编码，应从数据计算
- [ ] "中途醒来次数 0次" 为硬编码，应从数据计算
- [ ] "1小时11分钟 (13:35-14:46)" 小睡数据为硬编码

**3. 周视图问题**
- [ ] 柱状图使用 dayNames[idx % 7] 循环，而不是实际星期几
- [ ] 没有显示每根柱子的具体睡眠时长数值
- [ ] 柱状图高度计算可能不合理（最大值硬编码为10小时）

**4. 月视图问题**
- [ ] 日历显示30个方块，但未标注具体日期
- [ ] 缺少月份切换功能
- [ ] 没有显示上周/下周对比

**5. 无数据状态**
- [ ] 当 todayRecord 为 null 时返回 null，无空状态提示

**6. 无障碍问题**
- [ ] 按钮缺少 aria-label
- [ ] 图表缺少 role 和 aria-label

**7. CSS问题**
- [ ] 大量使用 !important 覆盖 Card 组件样式，影响可维护性
- [ ] 某些样式未考虑小屏幕设备

### 修复方案

**MODIFIED Requirements**

### Requirement: 日视图 - 数据与显示一致性
- 睡眠总时长标题应根据实际数据日期显示（昨天/今天/具体日期）
- 当无睡眠数据时，显示空状态卡片

### Requirement: 日视图 - 环形图完整性
- 环形图应包含清醒(awake)部分，使用灰色显示
- 各阶段时长应从实际数据计算

### Requirement: 日视图 - 指标卡片数据来源
- 所有指标数值应从 mockHistory 数据计算得出
- 移除硬编码的血氧、心率、呼吸频率等数据

### Requirement: 周视图 - 柱状图正确性
- 每根柱子应显示对应的星期几（根据实际日期）
- 应在柱子上方或下方显示具体睡眠时长数值

### Requirement: 月视图 - 日历功能
- 每个日历格子应显示具体日期（1-30）
- 添加月份切换按钮（上一月/下一月）

### Requirement: 空状态处理
- 当无睡眠数据时，显示友好的空状态提示
- 提供"暂无睡眠数据，请佩戴手表入睡"的提示

### Requirement: 无障碍支持
- 所有交互按钮添加 aria-label
- 图表添加 role="img" 和 aria-label

### Requirement: CSS优化
- 移除不必要的 !important
- 使用更具体的 CSS 选择器
- 优化响应式布局
