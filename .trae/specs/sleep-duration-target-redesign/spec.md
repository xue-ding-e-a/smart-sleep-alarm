# 目标睡眠时长重构 - Product Requirement Document

## Overview
- **Summary**: 重构首页闹钟设置卡片，将目标睡眠时长作为主要设置项，唤醒时间窗口作为辅助项。支持自定义目标睡眠时长，并提供基于睡眠周期（1.5小时）的推荐梯度选项。
- **Purpose**: 优化用户设置逻辑，从"设置唤醒时间"转变为"设置睡眠目标"，更符合睡眠科学理念，提升用户体验和产品专业度。
- **Target Users**: 使用智能唤醒闹钟的所有用户，特别是关注睡眠质量和睡眠周期的用户。

## Goals
- 目标睡眠时长成为主要设置项，视觉上更突出
- 支持自定义目标睡眠时长（小时+分钟）
- 提供基于睡眠周期（1.5小时）的推荐梯度：4.5h、6h、7.5h、8h、9h
- 唤醒时间窗口变为辅助设置，视觉层级降低
- 更新建议文案，科普睡眠周期知识

## Non-Goals (Out of Scope)
- 不修改睡眠数据展示页面
- 不修改闹钟计算逻辑（仅UI调整）
- 不修改个人页面和设置页面
- 不增加新的健康数据接入

## Background & Context
- 当前首页闹钟设置卡片以唤醒时间窗口（最早/最晚）为主，目标睡眠时长为辅
- 科学研究表明，睡眠以90分钟为一个周期，在周期结束时醒来会更加精神
- 将目标睡眠时长作为主设置项，更符合产品"AI睡眠最优唤醒"的定位
- 推荐梯度基于睡眠周期：4.5h(3个周期)、6h(4个)、7.5h(5个)、8h(常规推荐)、9h(6个)
- 8h 虽然不是精确的周期倍数，但作为常规健康推荐值保留

## Functional Requirements
- **FR-1**: 闹钟设置卡片中，目标睡眠时长区域置于唤醒时间窗口区域之前
- **FR-2**: 目标睡眠时长支持自定义输入（小时和分钟）
- **FR-3**: 提供5个推荐梯度快捷选项：4.5h、6h、7.5h、8h、9h
- **FR-4**: 点击推荐选项可快速设置目标睡眠时长
- **FR-5**: 唤醒时间窗口（最早/最晚）作为辅助设置保留
- **FR-6**: 建议文案更新为"建议以一个半小时为一个睡眠周期设置目标睡眠时长"

## Non-Functional Requirements
- **NFR-1**: 界面响应流畅，交互反馈及时
- **NFR-2**: 支持0.5小时（30分钟）为最小粒度的自定义时长
- **NFR-3**: 视觉层级清晰，目标睡眠时长比唤醒时间窗口更突出
- **NFR-4**: 保持整体设计风格与现有页面一致

## Constraints
- **Technical**: React + TypeScript + Vite，现有组件体系
- **Business**: 保持数据结构兼容，不破坏现有设置存储
- **Dependencies**: 依赖现有的 useAlarmSettings hook

## Assumptions
- 用户理解睡眠周期的概念，或通过建议文案可以理解
- 8小时作为通用推荐值，即使不是精确的周期倍数也应该保留
- 最小设置粒度为30分钟足够满足需求

## Acceptance Criteria

### AC-1: 布局顺序调整
- **Given**: 用户打开首页闹钟设置卡片
- **When**: 查看卡片内容布局
- **Then**: 目标睡眠时长区域在唤醒时间窗口区域之前，视觉上更突出
- **Verification**: `human-judgment`
- **Notes**: 目标时长区域应有更醒目的标题和更大的数值展示

### AC-2: 推荐梯度选项
- **Given**: 用户查看目标睡眠时长设置
- **When**: 查看快捷选项
- **Then**: 显示5个推荐选项：4.5h、6h、7.5h、8h、9h
- **Verification**: `programmatic`
- **Notes**: 8h为默认选中项

### AC-3: 自定义时长输入
- **Given**: 用户想要设置非推荐值的睡眠时长
- **When**: 用户使用自定义输入
- **Then**: 可以设置小时和分钟，最小粒度为30分钟
- **Verification**: `human-judgment`
- **Notes**: 支持步进器或输入框形式

### AC-4: 推荐选项点击生效
- **Given**: 用户在目标睡眠时长设置区域
- **When**: 点击任意推荐梯度选项
- **Then**: 该选项变为选中状态，目标睡眠时长更新为对应值
- **Verification**: `programmatic`

### AC-5: 唤醒时间窗口保留
- **Given**: 用户查看闹钟设置卡片
- **When**: 向下滚动/查看完整内容
- **Then**: 最早/最晚唤醒时间设置仍然存在，但视觉层级低于目标睡眠时长
- **Verification**: `human-judgment`

### AC-6: 建议文案更新
- **Given**: 用户查看目标睡眠时长区域
- **When**: 查看提示/建议文字
- **Then**: 显示"建议以一个半小时为一个睡眠周期设置目标睡眠时长"
- **Verification**: `programmatic`

### AC-7: 设置持久化
- **Given**: 用户修改了目标睡眠时长
- **When**: 刷新页面或重新进入
- **Then**: 设置值保持不变
- **Verification**: `programmatic`
